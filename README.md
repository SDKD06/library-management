# 📚 Library Management System

A full-stack Library Management System with an **Angular** frontend, a **Spring Boot (Java)** backend, and a **MySQL** database. It handles the everyday operations of a library — managing books, members, and borrow/return transactions — secured with JWT authentication and role-based access.

Beyond standard CRUD, it includes real operational **workflows** (book reservations and fine management) and an analytics layer with a **Book Demand Predictor** and **personalized recommendations**.

---

## ✨ Features

- **Authentication & Authorization** — JWT-based login with role-based access (`ADMIN`, `LIBRARIAN`).
- **Books** — Add, edit, delete, and list books (title, author, genre).
- **Members** — Manage library members and their status.
- **Transactions** — Borrow and return books, with live status tracking (`ACTIVE`, `COMPLETED`, `OVERDUE`).
- **Automatic Overdue Detection** — A scheduled job flags loans as `OVERDUE` once they pass the 14-day limit (runs at startup and daily at midnight).
- **Reservations Workflow** ⭐ — Members request books; a librarian approves or rejects each request (`PENDING → APPROVED / REJECTED`).
- **Fine Management** ⭐ — Overdue books accrue a ₹5/day fine; fines can be generated from overdue loans, then marked paid or waived.
- **Dashboard** — At-a-glance stats: total books, available, borrowed, overdue, members, and a genre breakdown.
- **Reports** — Most borrowed books, most active members, and overdue summaries.
- **Book Demand Predictor** ⭐ — Forecasts the next most-likely-to-be-borrowed books, with summary cards, a ranked list, and a genre-colored bar chart.
- **Personalized Recommendations** ⭐ — Suggests books for a specific member based on their own borrowing history and favourite genres.

## 🚀 Getting Started

### Prerequisites

- **Java 17+** (JDK)
- **Node.js 18+** and **Angular CLI** (`npm install -g @angular/cli`)
- **MySQL 8** running locally

### 1. Database setup

Create the database (all tables are auto-generated by Hibernate on first run):

```sql
CREATE DATABASE bookdb;
```

### 2. Configure the backend

Open `backend/src/main/resources/application.properties` and set your own MySQL credentials:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/bookdb
spring.datasource.username=YOUR_DB_USERNAME
spring.datasource.password=YOUR_DB_PASSWORD
spring.jpa.hibernate.ddl-auto=update

# JWT secret (use your own long, random string)
jwt.secret=YOUR_SECRET_KEY
```

> **Note:** `application.properties` is git-ignored so credentials are never committed. Each developer sets their own.

### 3. Run the backend

From the `backend/` folder:

```bash
# Windows (PowerShell)
.\mvnw spring-boot:run

# macOS / Linux
./mvnw spring-boot:run
```

The API starts on **http://localhost:8080**.

### 4. Run the frontend

From the `frontend/` folder:

```bash
npm install
ng serve
```

The app starts on **http://localhost:4200**.

## 🔑 Logging In

Users are stored in the `users` table with BCrypt-hashed passwords and a role (`ADMIN` or `LIBRARIAN`). Register a user through the auth endpoint (or seed one directly in the database), then log in via the UI at `http://localhost:4200`.

## 📡 API Overview

All endpoints are served from `http://localhost:8080`. Protected routes require a `Bearer <token>` header, which the Angular interceptor attaches automatically after login.

| Method | Endpoint                     | Description                                    |
| ------ | ---------------------------- | ---------------------------------------------- |
| POST   | `/auth/register`             | Register a new user                            |
| POST   | `/auth/login`                | Log in and receive a JWT                       |
| GET    | `/books`                     | List all books                                 |
| POST   | `/books`                     | Add a book                                     |
| PUT    | `/books/{id}`                | Update a book                                  |
| DELETE | `/books/{id}`                | Delete a book (ADMIN only)                     |
| GET    | `/members`                   | List members                                   |
| GET    | `/transactions`              | List transactions                              |
| POST   | `/transactions`              | Borrow a book                                  |
| GET    | `/reservations`              | List reservations                              |
| POST   | `/reservations`              | Request a book (creates a PENDING reservation) |
| PUT    | `/reservations/{id}/approve` | Approve a reservation                          |
| PUT    | `/reservations/{id}/reject`  | Reject a reservation                           |
| GET    | `/fines`                     | List fines                                     |
| POST   | `/fines/generate`            | Generate fines for overdue books               |
| PUT    | `/fines/{id}/pay`            | Mark a fine paid                               |
| PUT    | `/fines/{id}/waive`          | Waive a fine                                   |
| GET    | `/predictions?limit=`        | Top predicted books overall                    |
| POST   | `/predictions`               | Predictions biased toward selected genres      |
| GET    | `/predictions/member/{id}`   | Personalized recommendations for a member      |

> Exact sub-routes are defined in the controllers under `backend/.../controller/`.

## 🔮 The Analytics Features

### Book Demand Predictor

Estimates which books are most likely to be borrowed next. It is a **transparent, recency-weighted scoring model** (no machine learning) that runs over the existing transaction history.

Each book gets a score out of 100, blending three signals:

| Signal             | Weight | What it measures                                     |
| ------------------ | ------ | ---------------------------------------------------- |
| **Book momentum**  | 45%    | How much _this book_ has been borrowed recently      |
| **Genre momentum** | 35%    | How popular _its genre_ is right now                 |
| **Genre match**    | 20%    | A bonus if the book matches the genre you focused on |

**Recency matters most.** Every borrow is weighted so that recent activity counts more — a borrow's weight halves every 30 days. The predictor page shows summary cards, a genre-colored **demand forecast bar chart** (hand-built SVG, no chart library), and a ranked list where each book carries a plain-English reason (e.g., _"3 borrows in the last 30 days; Technology is trending"_).

### Personalized Recommendations

Suggests books for a **specific member** based on their own history. It reads which genres that member borrows most (recent borrows weighted higher), then recommends books in those genres they haven't read yet — blended **65% personal taste / 35% library-wide popularity**. Each recommendation shows a "% match" and a reason such as _"Because you enjoy Science books."_ Members with no history fall back gracefully to popular picks.

## 🔄 Workflows

### Reservations

A member requests a book → the request is `PENDING` → a librarian **approves** or **rejects** it, stamping a decision date. The page includes a request form and a table with a live pending count.

### Fines

Overdue loans accrue a ₹5/day fine. Clicking **Generate Fines** scans every `OVERDUE` transaction and creates a fine for any that doesn't have one (safe to run repeatedly). Each fine can be **marked paid** or **waived**, and summary cards show outstanding vs. collected totals.
